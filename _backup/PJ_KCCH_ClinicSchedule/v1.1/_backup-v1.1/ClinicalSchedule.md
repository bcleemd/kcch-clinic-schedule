# ClinicalSchedule.md

이 문서는 `json_to_html.py` 스크립트가 생성하는 웹 파일들(`index.html`, `style.css`, `script.js`)의 상호작용 방식과 데이터 변경 시 관리 요령에 대해 설명합니다.

---

## 1. 파일별 역할 및 데이터 종속성

결론적으로, JSON 내용이 바뀌었을 때 반드시 새로 생성되어야 하는 것은 **`index.html`**뿐입니다. `style.css`와 `script.js`는 디자인과 동작 로직을 담당하므로 데이터 내용에 관계없이 재사용이 가능합니다.

### A. index.html (매번 새로 생성 필수)
*   **이유**: 진료과목 리스트(필터 체크박스)와 실제 의사 명단, 시간표 데이터가 이 파일 안에 직접 HTML 태그로 포함되어 있기 때문입니다.
*   **역할**: 화면 구조를 정의하고 실제 데이터를 보관합니다.

### B. style.css (재사용 가능)
*   **이유**: 이 파일은 표의 색상, 굵은 선의 두께, 레이아웃 등 **"디자인 규칙"**만 담고 있습니다. 
*   **특징**: 진료과가 늘어나거나 의사 이름이 바뀌어도 "표는 이렇게 그려라"라는 규칙은 변하지 않습니다.

### C. script.js (재사용 가능)
*   **이유**: 이 파일은 **"체크박스를 누르면 해당 이름의 행을 숨겨라"**라는 **"동작 로직"**만 담고 있습니다.
*   **유연성**: 특정 진료과 이름을 코드로 고정한 것이 아니라, 화면에 있는 모든 체크박스를 동적으로 찾아 동작하도록 설계되었습니다. 따라서 JSON에 새로운 진료과가 추가되어도 JS 파일 수정 없이 자동으로 인식합니다.

---

## 2. 관리 및 수정 가이드

`json_to_html.py` 스크립트는 매 실행 시 세 파일을 모두 새로 작성하도록 되어 있습니다. 이는 프로젝트를 스크립트 하나로 완벽하게 복구(Self-healing)하기 위한 설계입니다.

### 상황별 대응 가이드
1.  **데이터만 바뀐 경우**
    *   `uv run json_to_html.py`를 실행하면 됩니다.
2.  **디자인이나 기능을 영구적으로 고치고 싶은 경우**
    *   `index.html`, `style.css`, `script.js` 파일을 직접 수정하지 마세요.
    *   대신 `json_to_html.py` 파이썬 코드 내의 `css_content` 또는 `js_content` 변수 문자열을 수정해야 합니다. 
    *   그래야 나중에 데이터가 업데이트되어 다시 생성될 때도 사용자가 수정한 디자인/기능이 유지됩니다.
