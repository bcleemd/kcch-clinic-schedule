# json_to_html.py 상세 설명 및 실행 과정

이 문서는 `json_to_html.py` 스크립트의 내부 동작 원리와 실행 과정, 그리고 최근 업데이트된 반응형 최적화 기능에 대해 상세히 설명합니다.

---

## 1. 실행 순서 및 동작 원리

스크립트가 실행되면 다음과 같은 일련의 과정을 거쳐 웹페이지를 생성합니다.

### Step 1: 입력 및 환경 설정 (Setup)
- 명령행 인자(sys.argv)를 통해 대상 JSON 파일 경로를 입력받습니다.
- 실행되는 위치를 기준으로 출력이 저장될 경로(`index.html`, `style.css`, `script.js`)를 설정합니다.

### Step 2: 데이터 로드 및 전처리 (Data Processing)
- JSON 파일을 열어 `schedule_info`(제목, 업데이트 날짜)와 `departments`(진료과별 데이터)를 추출합니다.
- 추출된 데이터에서 진료과 목록만 모아 하단 필터(Checkbox) 생성을 위한 리스트를 준비합니다.

### Step 3: 의료진 색상 맵핑 (Color Management)
- 각 의사 이름별로 고정된 고대비 색상을 할당합니다.
- `김진(초음파)`와 같이 괄호가 포함된 경우, `김진`이라는 이름만 추출하여 동일 인물에게 같은 색상이 부여되도록 처리합니다.

### Step 4: HTML 구조 생성 (Generation)
- **표(Table) 본문 생성**: 진료과별로 오전(AM)행과 오후(PM)행을 생성합니다.
- **반응형 텍스트 처리**: 모바일 화면 최적화를 위해 특정 텍스트 구간에 `<span class="mobile-block">` 태그를 삽입합니다.
- **필터 생성**: JSON에 포함된 모든 진료과에 대해 검색/필터링용 체크박스를 생성합니다.

### Step 5: 리소스 파일 저장 (Output)
- 준비된 CSS 템플릿과 JS 로직, 그리고 완성된 HTML 구조를 각각의 파일로 저장합니다.

---

## 2. 주요 로직 상세 설명

### A. 의료진 이름 및 직책 분리 (최근 업데이트)
모바일 기기의 좁은 화면을 위해 이름과 직책을 지능적으로 분리합니다.
- **로직**: 이름에 괄호(`(`)가 포함되어 있으면, 괄호 앞부분(이름)과 괄호 뒷부분(직책/용도)을 나눕니다.
- **결과**: 데스크탑에서는 한 줄로 보이지만, 모바일에서는 직책 부분이 `display: block`으로 변하며 자동으로 줄바꿈됩니다.

### B. 오전/오후 텍스트 최적화 (최근 업데이트)
표의 좌측 '오전', '오후' 텍스트 셀의 가로 폭을 줄이기 위해 적용되었습니다.
- **로직**: `오`와 `전/후` 사이에 `mobile-block` 스팬을 삽입합니다.
- **효과**: 모바일 화면에서만 세로로 길게 배치되어 표의 가로 공간을 더 확보해 줍니다.

### C. 동적 필터링 시스템 (JavaScript)
- 모든 체크박스에 이벤트 리스너를 등록합니다.
- 체크 상태가 변할 때마다 현재 선택된 진료과 목록을 확인하고, 해당하지 않는 행(`tr[data-dept]`)을 `display: none`으로 숨깁니다.
- '전체 선택/해제' 기능을 통해 편리한 조작을 지원합니다.

---

## 3. 관리자 가이드

`json_to_html.py`는 **"단일 진실 공급원(Single Source of Truth)"** 방식으로 설계되었습니다.

1. **디자인 변경**: 스크립트 내의 `css_content` 변수 문자열을 수정하십시오.
2. **동작 로직 변경**: 스크립트 내의 `js_content` 변수 문자열을 수정하십시오.
3. **영속성**: 생성된 HTML/CSS/JS 파일을 직접 수정하면, 다음에 데이터를 업데이트(`uv run`)할 때 수정한 내용이 덮어씌워집니다. 따라서 모든 변경 사항은 `json_to_html.py`에 반영하는 것을 권장합니다.
