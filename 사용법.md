# 외래진료일정표 편집기 및 배포 시스템 사용법

이 가이드는 외래진료일정표를 시각적으로 편집하는 WYSIWYG 에디터의 실행 방법과, 수정된 시간표 데이터를 GitHub 원격 저장소에 안전하고 빠르게 배포하는 방법 및 규칙을 설명합니다.

---

## 1. 프로젝트 핵심 요약

* **서버 단일화 및 자동화 (포트 5050):** 로컬 환경에서 단일 Express 서버가 뷰어와 에디터를 동시에 서빙하며, 서버 실행 폴더를 자동 스캔하여 데이터를 자동으로 로딩합니다.
* **이중 저장 (Dual-Save):** 에디터에서 저장 시 메인 서비스 파일(`clinic-schedule.json`)이 갱신되며, 동시에 연월 백업본(`clinic-schedule-YYYYMM.json`)이 현재 폴더에 자동 생성됩니다.
* **GitHub 다이렉트 배포 (Deploy to GitHub):** 뷰어가 동적으로 `clinic-schedule.json`을 읽어 렌더링하기 때문에 별도 HTML 빌드가 필요 없습니다. 에디터에서 `[Deploy to GitHub]` 버튼을 누르면 `gh` CLI 툴을 통해 원격 저장소의 `clinic-schedule.json` 파일만 신속하게 직접 푸시(업데이트)됩니다.

---

## 2. 에디터 실행 및 접속 방법

### A. 로컬 서버 실행
로컬 터미널에서 아래 명령어를 실행하여 서버를 가동합니다.
```bash
npm run dev
```
* **동작:** nodemon 기반의 Express 서버가 포트 **`5050`**에서 구동을 시작합니다.
* **접속 주소**:
  * **진료시간표 뷰어 (Viewer)**: [http://localhost:5050/](http://localhost:5050/)
  * **진료시간표 편집기 (Editor)**: [http://localhost:5050/editor_home](http://localhost:5050/editor_home)

### B. 폴더 자동 스캔 및 데이터 자동 로드
* 에디터(`http://localhost:5050/editor_home`)에 처음 접속하면, 브라우저에서 폴더를 선택할 필요 없이 서버가 실행된 디렉토리(`KCCH-ClinicSchedule`)를 자동 스캔하여 `clinic-schedule.json` 데이터를 화면에 바로 띄워줍니다.

---

## 3. 데이터 편집 및 저장/배포 과정

에디터 화면에서 의사 카드를 스케줄 셀로 드래그하여 배치를 완료한 후 아래 단계를 거쳐 배포합니다.

### 1단계: [JSON 저장] 버튼 클릭
* 편집한 스케줄이 로컬 디렉토리의 **`clinic-schedule.json`**에 덮어씌워져 저장됩니다.
* 동시에, 입력된 연월 메타데이터에 맞춰 **`clinic-schedule-YYYYMM.json`** 파일이 자동으로 이중 저장(Dual-Save)되어 백업본이 유지됩니다.
* *작업 흐름 편의성을 위해 저장이 완료되더라도 에디터는 백업 파일로 타겟을 변경하지 않고 메인 파일(`clinic-schedule.json`) 상태를 유지합니다.*

### 2단계: [Deploy to GitHub] 버튼 클릭
* **배포 작동 방식**: 백엔드 서버가 GitHub CLI(`gh` api)를 실행하여 원격 저장소의 기존 파일 SHA를 동적으로 탐색하고, 로컬의 최신 `clinic-schedule.json` 파일 데이터만을 깃허브 저장소에 직접 API를 통해 커밋/푸시합니다.
* **선행 조건**: 배포를 수행하기 전, 컴퓨터 터미널에서 **딱 한 번** 아래 명령어를 실행하여 깃허브 자격증명 로그인을 마쳐야 합니다:
  ```bash
  gh auth login
  ```
  *(안내에 따라 브라우저에서 1회 인증 로그인을 마치면 연동이 완료됩니다.)*

## 4. 진료시간표 뷰어 (Viewer) 작동 원리

진료시간표 뷰어([http://localhost:5050/](http://localhost:5050/))는 서버 연동과 웹 브라우저 최적화 렌더링을 위해 아래와 같은 클라이언트-사이드 로직을 통해 실행됩니다.

### A. 동적 데이터 바인딩
* `index.html`이 브라우저에 로드되면 `script.js`의 `init()` 함수가 실행됩니다.
* `fetch('clinic-schedule.json', { cache: 'no-store' })`를 호출하여 최신 스케줄 데이터를 캐시 없이 가져오고, 브라우저 메모리상에서 즉시 렌더링(Client-Side Rendering)하여 테이블에 바인딩합니다. (이를 통해 별도의 빌드 과정 없이 데이터 갱신만으로 실시간 반영됩니다.)

### B. 스마트 의료진 고유 색상 매핑
* `getDoctorColor(name)` 함수를 통해 12종의 고대비/고선명 프리미엄 색상 팔레트가 의료진 이름별로 자동 매핑됩니다.
* 괄호가 포함된 의료진 명칭(예: `임의사(초음파)`)의 경우, 괄호 앞의 본명(`임의사`)만을 추출하여 동일 인물에게는 요일이나 오전/오후 구분과 상관없이 일관되게 동일한 배경 글씨 색상이 할당되도록 처리되었습니다.

### C. 반응형 공간 활용 및 모바일 최적화 Layout
* **구분 셀 너비 압축**: 모바일 화면에서 한정된 가로 공간을 극대화하기 위해 '오전/오후' 텍스트에 `<span class="mobile-block">` 처리를 적용했습니다. 모바일에서는 '오'와 '전/후' 사이에 줄바꿈이 발생하여 셀의 가로폭이 대폭 줄어들고 스케줄 가로 영역이 늘어납니다.
* **의료진 명칭 개행**: 의료진 카드도 모바일 화면에서는 이름과 괄호 안의 직책/용도(예: `(초음파)`)가 위아래로 줄바꿈되어, 좁은 너비에서도 텍스트가 찌그러지지 않고 깔끔하게 표출됩니다.
* **월/수/금 시인성 강화**: 월/수/금 요일 컬럼에 뚜렷한 파스텔톤 배경색(오전: `#d2e9c4`, 오후: `#b5dba0`)을 지정하여, 회색/파란색 톤의 화/목 요일과 직관적으로 대비를 이루어 요일 오인을 방지합니다.

### D. 실시간 진료과 필터링 시스템
* 데이터 내에 존재하는 모든 진료과(`dept_name`)를 동적으로 추출하여 하단 필터 컨트롤러 영역에 체크박스 리스트를 생성합니다.
* `전체 선택` 및 `전체 해제` 컨트롤러를 내장하고 있으며, 체크박스 이벤트가 변경될 때마다 Javascript가 실시간으로 매칭되지 않는 진료과 행(`tr[data-dept]`)의 `display` 스타일을 제어하여 부드럽고 끊김 없는 필터링 경험을 제공합니다.

## 5. 수동 빌드 및 배포 방법 (CLI 직접 사용 시)

에디터 웹 화면을 통하지 않고 터미널에서 CLI로 수동 빌드/배포를 실행해야 할 때는 아래 명령어를 사용합니다.

### 1) Python을 이용한 정적 HTML 뷰어 빌드
기존 정적 뷰어 빌드가 필요한 경우 Python 환경에서 스크립트를 수행합니다. (`pyproject.toml` 감지에 따라 `uv`를 사용해야 합니다.)
```bash
uv run json_to_html.py clinic-schedule.json
```
* **결과**: `clinic-schedule.json`을 파싱하여 정적 `index.html`, `style.css`, `script.js`를 프로젝트 루트에 빌드합니다.

### 2) GitHub에 수동 Push
```bash
git add clinic-schedule.json
git commit -m "feat: manual update clinic schedule data"
git push origin main
```
